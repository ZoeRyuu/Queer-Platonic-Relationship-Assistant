# QPR 助手 — 产品需求文档 (PRD)

> 一个纯本地的酷儿柏拉图关系(Queerplatonic Relationship)自我探索与关系管理网页应用。
> 本文档面向 Claude Code,可直接作为开发依据。

---

## 1. 产品概述

**产品名称**(暂定): QPR 助手 / Kinship(可后续再定)

**一句话描述**: 帮助用户理清自己对非传统关系的需求与期待,并记录、管理自己的酷儿柏拉图关系。

**目标用户**: 所有对非传统关系(QPR、柏拉图伴侣、承诺友谊等)感兴趣的人,不限性向。

**核心原则**:
1. **隐私至上**: 所有数据仅存储在用户浏览器本地(IndexedDB),不上传任何服务器,无账号系统,无埋点统计。
2. **纯工具型**: 不接入 AI,所有引导问题、模板均预设在应用内。
3. **不评判**: 文案和交互不预设任何关系"应该"是什么样,不使用恋爱关系的默认框架。

---

## 2. 技术要求

- **形态**: 单页网页应用 (SPA),支持 PWA(可安装到手机主屏、离线可用)
- **技术栈建议**: React + Vite + TypeScript + Tailwind CSS
- **数据存储**: IndexedDB(建议用 Dexie.js 封装),不使用任何后端
- **数据安全**:
  - 提供"导出全部数据为 JSON 文件"和"从 JSON 导入"功能(备份与迁移)
  - 提供"一键清除全部数据"功能(需二次确认)
  - 可选: 应用锁(本地 PIN 码,仅作轻量遮挡,需说明不是加密)
- **响应式**: 手机优先设计,桌面端良好适配
- **语言**: 简体中文界面

## 3. 功能规划(分三期)

### 第一期 MVP: 自我探索模块 ⭐ 优先开发

帮助用户想清楚"我想要什么样的关系"。

#### 3.1 引导式问卷("关系需求探索")

结构化的自我探索流程,分为若干主题章节,用户可分次完成,进度自动保存:

**章节 A — 亲密的形状**
- 对我来说,"亲密"意味着什么?(多选+自由文本: 深度对话 / 身体接触 / 共同生活 / 共享日常 / 相互照顾 / 精神共鸣…)
- 我需要多少独处空间?(滑块 + 说明文本)
- 哪些形式的身体接触让我舒适 / 不舒适?(可勾选: 拥抱、牵手、依偎、无身体接触…每项可标注"舒适/视情况/不适")

**章节 B — 承诺与期待**
- 我希望关系中包含哪些承诺?(共同居住 / 财务互助 / 紧急联系人 / 长期陪伴 / 共同抚养宠物或孩子 / 不预设…)
- 我对"排他性"的态度?(情感排他 / 不要求排他 / 需要协商 / 还不确定)
- 这段关系在我生活中的优先级期待?

**章节 C — 边界与雷区**
- 什么行为会让我感到被侵犯边界?(自由文本 + 常见选项)
- 我在关系中最害怕什么?
- 过往经历中,什么让我感到安全 / 不安全?

**章节 D — 沟通风格**
- 发生分歧时,我倾向于?(立即沟通 / 冷静后再谈 / 书面表达 / 需要对方主动…)
- 我表达关心的方式 / 我希望被关心的方式(参考"爱之语"但去浪漫化的版本: 陪伴时间 / 肯定言语 / 实际帮助 / 礼物 / 身体接触)

#### 3.2 探索结果页("我的关系画像")

- 完成问卷后自动生成一页可视化摘要: 我的亲密需求、边界清单、承诺期待、沟通偏好
- 支持随时修改单项答案,画像实时更新
- 支持导出为图片或纯文本(方便分享给关系中的对方——由用户自己决定是否分享)
- 支持创建多份画像快照,对比自己不同时期的变化

#### 3.3 概念小百科(轻量)

- 内置 10–15 个词条: QPR、无浪漫倾向(aromantic)、无性恋(asexual)、squish、zucchini、关系无政府主义(RA)、承诺友谊等
- 每个词条 100–300 字,语气中立友好
- 在问卷中出现相关术语时可点击查看

---

### 第二期: 关系档案

- 为每段关系建立档案: 称呼、关系性质(自定义标签,不预设"伴侣/朋友"二分)、开始时间、当前状态
- 每份档案记录: 双方约定的边界、承诺内容、重要纪念日、共同约定(自由文本 + 结构化字段)
- 边界和承诺支持"版本历史"——关系是流动的,修改时保留旧版本记录
- 档案支持归档(关系结束不等于删除记忆)

### 第三期: 沟通工具

- **边界协商模板**: 预设话题清单(身体接触、时间分配、对外介绍方式、财务、排他性…),生成一份可以和对方一起过一遍的"协商议程",支持记录协商结果直接写入关系档案
- **定期回顾提醒**: 用户自定周期(如每3个月),到期时应用内提醒做一次关系回顾,提供回顾问题模板(哪些约定运行良好?哪些需要调整?),回顾记录存入档案
- 提醒基于 PWA 本地通知,不依赖服务器

## 4. 页面结构 (MVP)

```
├── 首页(引导语 + 三大模块入口,第二三期模块显示"即将推出")
├── 自我探索
│   ├── 章节列表(显示各章节完成进度)
│   ├── 答题页(每屏一题,可跳过,可回退)
│   └── 我的关系画像(结果页)
├── 小百科(词条列表 + 详情)
└── 设置
    ├── 数据导出 / 导入
    ├── 清除全部数据
    └── 关于本应用(隐私说明: 数据只在你的设备上)
```

## 5. 设计基调

- 温暖、中性、不粉红不彩虹刻板印象;建议低饱和暖色系
- 文案避免"恋人""爱情"等默认浪漫框架词汇,用"关系""亲密""联结"
- 空状态和引导文案要有陪伴感,但不说教
- 无障碍: 合理的对比度、支持键盘操作

## 6. 明确不做的事

- ❌ 社交、匹配、任何用户间互动
- ❌ 账号系统、云端同步、数据上传
- ❌ AI 功能
- ❌ 任何形式的数据统计和追踪

## 7. 开发顺序建议(给 Claude Code 的分步指令)

1. 初始化项目: Vite + React + TS + Tailwind + Dexie,搭好路由和页面骨架
2. 实现 IndexedDB 数据层: 问卷答案的存取、进度保存
3. 实现自我探索问卷: 章节 A–D 的题目组件(单选/多选/滑块/文本)
4. 实现"关系画像"结果页
5. 实现小百科词条页(词条内容写死在代码里即可)
6. 实现设置页: JSON 导出/导入、清除数据
7. PWA 配置(manifest + service worker 离线缓存)
8. 视觉打磨与响应式适配
